## Mohini — The Magnum Opus

<div align="center">

[![mohini](https://github-readme-stats.vercel.app/api/pin/?username=darshjme&repo=mohini&theme=github_dark&border_color=30363d&title_color=58A6FF&icon_color=F78166&text_color=8b949e)](https://github.com/darshjme/mohini)

</div>

```
mohini/
├── 14-crate Rust workspace        — zero-copy, memory-safe, production-grade
├── 104 bundled skill categories   — skills as WASM modules, hot-swappable
├── 40 channel adapters            — Slack, WhatsApp, Telegram, email, API, ...
├── 188 model integrations         — local + cloud, OpenAI, Anthropic, Ollama, ...
├── WASM sandbox runtime           — untrusted skill execution, isolated
├── Self-healing orchestration     — agents recover, retry, re-route autonomously
└── Zero-downtime deployment       — rolling updates, no cold starts
```

Mohini is not a framework wrapper. It is a **ground-up agentic operating system** — a runtime where agents have persistent memory, composable skills, multi-channel presence, and the ability to spawn, coordinate, and hand off to specialist sub-agents.

**Core architecture decisions:**
- **Rust throughout** — ownership model eliminates entire classes of agent runtime bugs
- **Tiered memory** — hot cache (Redis) + vector store (Qdrant) + persistent knowledge base, with semantic recall and compaction
- **Skill-as-WASM** — every skill is a sandboxed module; untrusted code cannot compromise the host
- **Shadow spawning** — orchestrator spawns specialist sub-agents for missions, hands off via KB, next spawn inherits full context
- **Self-healing** — agents detect failure states and re-route without human intervention
